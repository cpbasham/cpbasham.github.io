Enumerable Methods

Map

6/18/15

Map is one higher-order function in what many people consider the functional programming trifecta (map, filter, fold). In Ruby this method is known as both Enumerable#map and Enumerable#collect (the 'collect' alias coming from the language Smalltalk). The function takes in a list of values (in this case an Enumerable) and returns a copy of the list of values, with the given block applied to each element in the Enumerable. For example, you can easily square every number in an enumerable this way:
(1..5).map{ | i | i * i }=> [1, 4, 9, 16, 25]
This is especially useful when we want to create a modified copy of an array. It is also probably the most elegant way to apply some function to every element in an Enumerable (using map!):
x = ["one", "two", "three", "four", "five"]
x.map!{ | s | s.upcase }
puts x.to_s=> ["ONE", "TWO", "THREE", "FOUR", "FIVE"]
Note that while #map and #each can be considered similar in some ways, #map returns a new array, whereas #each (along with #map!) returns the same array.
